Ya no se usan para maquetar la web, las **tablas** siguen siendo vitales para mostrar datos estructurados. Aquí va la receta completa:

---

|Elemento|¿Para qué sirve?|Render por defecto|
|---|---|---|
|`<table>`|Contenedor principal.|Borde colapsado y celdas alineadas arriba-izquierda.|
|`<caption>`|Título de la tabla (primer hijo de `<table>`).|Centrado encima.|
|`<thead>` / `<tbody>` / `<tfoot>`|Cabecera, cuerpo y pie.|Agrupan filas; utilísimo para orden, estilo y accesibilidad.|
|`<tr>`|Fila.|Nueva línea.|
|`<th>`|Celda de encabezado.|Texto en **negrita** y centrado.|
|`<td>`|Celda de datos.|Texto normal, alineado a la izquierda.|
|`<colgroup>` / `<col>`|Definir grupos o columnas para estilos generales.|Sin estilo visible.|

---

#### Ejemplo

```html
<table>
  <caption>Ventas trimestrales 2025</caption>
  <colgroup>
    <col span="1" style="background:#f2f2f2">
    <col span="3">
  </colgroup>

  <thead>
    <tr>
      <th scope="col">Producto</th>
      <th scope="col">Q1</th>
      <th scope="col">Q2</th>
      <th scope="col">Q3</th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <th scope="row">Camisetas</th>
      <td>1 020 €</td><td>980 €</td><td>1 150 €</td>
    </tr>
    <tr>
      <th scope="row">Sudaderas</th>
      <td colspan="2">— sin stock —</td><td>630 €</td>
    </tr>
  </tbody>

  <tfoot>
    <tr>
      <th scope="row">Total</th>
      <td>1 020 €</td><td>980 €</td><td>1 780 €</td>
    </tr>
  </tfoot>
</table>

```

- **`<caption>`** describe la tabla para todos, incluidos screen-readers.
    
- **`scope="col"` / `scope="row"`** vincula cabeceras con datos; accesibilidad impecable.
    
- **`colspan` / `rowspan`** fusionan celdas para agrupaciones lógicas.
    

---

#### Atributos clave (pocos y buenos)

|Atributo|Etiqueta|Valor típico|Comentario|
|---|---|---|---|
|`scope`|`<th>`|`row`, `col`, `rowgroup`, `colgroup`|Relación cabecera-datos; fundamental para lectores de pantalla.|
|`colspan`|`<td>`, `<th>`|Número|Celdas que ocupa horizontalmente.|
|`rowspan`|`<td>`, `<th>`|Número|Celdas que ocupa verticalmente.|
|`border` (obsoleto)|`<table>`|`1`|**Ni se te ocurra**: usa CSS (`border`).|
|`width`, `height` (obsoleto)|Cualquiera|px, %|CSS también, gracias.|

---

#### Buenas prácticas

1. **Usa tablas solo para datos tabulares.** Para layout existen Flexbox y Grid.

2. **Define `<thead>`, `<tbody>` y `<tfoot>` siempre que la tabla pase de dos-tres filas.** Facilita scripts de ordenación y estilos.

3. **Añade `<caption>` descriptivo.** Mejor para SEO y accesibilidad.

4. **No te olvides de `scope` o `headers/id`**: sin ello, los lectores de pantalla “pierden” la referencia de cada celda.

5. **Hazlas “responsive” con CSS, no con scroll horizontal fijo:**
    ```css
    .tabla-scroll { overflow-x:auto; display:block; }
    ```

6. **Evita anidar tablas.** Impacta en rendimiento y complejidad.

7. **Estiliza con `border-collapse`, `text-align`, `nth-child()`.** Nada de atributos HTML heredados de los 90.


Esto es todo lo necesario para hacer tablas limpias, accesibles y fáciles de mantener.
