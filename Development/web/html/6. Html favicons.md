El **favicon** ( _favorite icon_ ) es el pequeño icono que identifica tu sitio en pestañas, marcadores, historial, accesos directos de móvil, resultados de búsqueda, etc. Aunque empezó como un simple `favicon.ico` de 16×16 px, hoy implica varios tamaños y formatos para adaptarse a navegadores y dispositivos.

---

#### 1. Dónde se declara

Pon los enlaces en el `<head>` de **todas** tus páginas o usa una plantilla común:
```html
<!-- Icono por defecto (fallback) -->
<link rel="icon" href="/favicon.ico" sizes="any">

<!-- PNG 32×32 para navegadores modernos -->
<link rel="icon" type="image/png" sizes="32x32" href="/icon-32.png">

<!-- PNG 16×16 por compatibilidad pura -->
<link rel="icon" type="image/png" sizes="16x16" href="/icon-16.png">

<!-- iOS / iPadOS Springboard -->
<link rel="apple-touch-icon" sizes="180x180" href="/apple-touch-icon.png">

<!-- Safari (macOS) modo oscuro/claro -->
<link rel="mask-icon" href="/safari-pinned-tab.svg" color="#5bbad5">

<!-- PWA & Android Chrome -->
<link rel="manifest" href="/site.webmanifest">

<!-- Windows 10/11 Tiles (opcional) -->
<meta name="msapplication-TileColor" content="#2d89ef">
<meta name="msapplication-config" content="/browserconfig.xml">

```

---

#### 2. Atributos y valores importantes

| Atributo | ¿Para qué sirve?                             | Valor típico                                             |
| -------- | -------------------------------------------- | -------------------------------------------------------- |
| `rel`    | Tipo de recurso.                             | `icon`, `apple-touch-icon`, `mask-icon`, `shortcut icon` |
| `href`   | Ruta al archivo.                             | `/favicon.ico`                                           |
| `type`   | MIME del icono.                              | `image/png`, `image/x-icon`, `image/svg+xml`             |
| `sizes`  | Dimensión exacta.                            | `16x16`, `32x32`, `180x180`, `any`                       |
| `color`  | Color de relleno para SVG Safari Pinned-Tab. | `#rrggbb`                                                |

_Si omites `type`, muchos navegadores lo infieren por la extensión. Si omites `sizes`, algunos asumen 16×16 (no recomendable)._

---
#### 3. Formatos recomendados

- **ICO**: contiene varias resoluciones en un solo archivo; Windows y viejos IE lo piden.

- **PNG**: comprimido sin pérdidas, alfa real; ideal para la mayoría de navegadores de escritorio y móvil.

- **SVG**: escalable; perfecto para modo oscuro/claro (Safari pinned-tab).

- **Web Manifest**: JSON que declara iconos para aplicaciones web progresivas (Android, Chrome OS).

---

#### 4. Tamaños que suelen cubrir todo

|Dispositivo / Uso|Tamaño|Formato sugerido|
|---|---|---|
|Pestaña de escritorio|32×32|PNG|
|Barra de marcadores|16×16|PNG o ICO|
|iOS Home Screen|180×180|PNG|
|Android Splash Screen (PWA)|512×512|PNG|
|Windows Tile (Start Menu)|150×150|PNG|

Genera todas de un _master_ a 1024×1024 px para evitar pixelaciones.

---

#### 5. Buenas prácticas, sin rodeos

1. **Almacena los iconos en la raíz (`/`)**; así los servidores y crawlers los encuentran fácil.

2. **Incluye al menos ICO 48×48 y PNG 32 / 180 / 512**; con eso cubres el 99 %.

3. **Usa SVG + `mask-icon`** para un favicon que cambie de color según el tema del sistema.

4. **Define `theme-color`** (`<meta name="theme-color" content="#ffffff">`) para que la UI de móviles combine con tu marca.

5. **Comprueba el orden**: los navegadores suelen usar el **primer** `rel="icon"` compatible que encuentren.

6. **Cachea agresivo** (`Cache-Control: max-age=31536000`) y cambia el nombre del archivo cuando actualices ( _cache-busting_ ).


Con esto tendrás favicons que se ven nítidos en cualquier dispositivo y transmiten tu marca en un vistazo.
