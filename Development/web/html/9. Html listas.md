Tres tipos de listas:

|Tipo|Etiqueta|¿Para qué sirve?|Render por defecto|
|---|---|---|---|
|**No ordenada**|`<ul>`|Agrupar ítems sin secuencia explícita.|Viñetas.|
|**Ordenada**|`<ol>`|Pasos, rankings, instrucciones.|Números.|
|**De definición**|`<dl>`|Termino-definición, glosarios, FAQs.|Bloque con sangría.|

---

#### Ejemplos

```html
<!-- Lista no ordenada -->
<ul>
  <li>Pan</li>
  <li>Leche</li>
  <li>Huevos</li>
</ul>

<!-- Lista ordenada, numeración romana desde el 3 y en reversa -->
<ol type="I" start="3" reversed>
  <li>Planificar</li>
  <li>Diseñar</li>
  <li>Programar</li>
</ol>

<!-- Lista de definición -->
<dl>
  <dt>HTML</dt>
  <dd>Lenguaje de marcado para la web.</dd>

  <dt>CSS</dt>
  <dd>Hojas de estilo en cascada.</dd>
</dl>

```

---

#### Atributos clave (solo los que cuentan)

|Etiqueta|Atributo|Valor típico|Efecto|
|---|---|---|---|
|`<ol>`|`type`|`1`, `A`, `a`, `I`, `i`|Cambia el marcador (números, letras mayús./minús., romanos).|
|`<ol>`|`start`|Número|Define el índice inicial (útil para listas divididas).|
|`<ol>`|`reversed`|(booleano)|Invierte la numeración (cuenta atrás).|
|`<ul>`|`type` (obsoleto)|`disc`, `circle`, `square`|**No lo uses**: controla las viñetas con CSS.|
|`<li>`|`value`|Número|Sobrescribe el contador de ese ítem concreto.|
|`<dl>`|—|—|Usa siempre `<dt>` (término) y `<dd>` (definición).|

---

#### Buenas prácticas que NO son opcionales

1. **Semántica ante todo.** Usa `<ul>` para menús, `<ol>` para pasos; no mezcles por estética.

2. **Viñetas y numeraciones = CSS.** Pasa de `type` en `<ul>` y dale estilo con `list-style-type` (`square`, `none`, etc.).

3. **Anidar sin miedo, pero con orden.** Cada nivel adicional va dentro de un `<li>` del nivel anterior.
	```html
	<ul>
	  <li>Frontend
	    <ul>
	      <li>HTML</li>
	      <li>CSS</li>
	    </ul>
	  </li>
	</ul>
	
	```
4. **No uses `<br>` para separar ítems.** Eso es barbarie; cada ítem va en su `<li>`.

5. **Accesibilidad automática.** Screen-readers indican el tamaño y la posición de la lista; no destruyas la semántica con `display: flex` sin añadir `role="list"`.

6. **Espaciado vía CSS, nunca HTML.** Nada de `&nbsp;` para sangrar: usa `margin` o `padding`.

7. **`<dl>` para FAQ reales, no para todo.** Si el par _término-definición_ no tiene sentido, usa otro contenedor.


Estas son las reglas para controlar las listas con: orden, claridad y semántica impecable.
